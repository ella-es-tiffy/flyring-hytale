## Project Overview

This is the **FlyRing mod** (branded as "IllegalRings" in-game) for Hytale - a Hytale server mod that adds five magical rings with unique abilities. The mod demonstrates advanced ECS-based gameplay mechanics and Hytale plugin architecture.

### What This Mod Does

Adds five craftable rings that players can equip:

1. **Fly Ring** - Grants creative-mode flight with auto-revocation when ring is removed
2. **Fire Ring** - Immunity to fire, lava, magma, and burn damage
3. **Water Ring** - Immunity to drowning and suffocation
4. **Heal Ring** - Lifesteal mechanic (configurable percentage, default 35%)
5. **Peaceful Ring** - Complete NPC immunity (mobs ignore and don't retaliate)

All rings are configurable via `mods/tiffy/config.json` without recompiling.

## Build System

### Building the Mod

```bash
cd _ModDevelopment/scripts
bash build_mod.sh FlyRing
```

**Build process:**
1. Compiles all Java files in `src/` with OpenJDK 21
2. Creates JAR containing compiled classes + resources
3. Extracts metadata from `resources/manifest.json` for JAR name
4. Prompts to install to `../../mods/` directory
5. Output: `FlyRing-{VERSION}.jar`

### Build Dependencies

- **Java Compiler**: `/opt/homebrew/opt/openjdk@21/bin/javac`
- **Hytale Server JAR**: `/Users/tiffy/Library/Application Support/Hytale/install/pre-release/package/game/latest/Server/HytaleServer.jar`
- **Installation Directory**: `../../mods/` (relative to scripts folder)

The build script automatically detects and includes `HytaleEnergy-*.jar` in the classpath if present (for future energy system integration).

## Architecture & Design Patterns

### Plugin Entry Point (IllegalRings.java)

The `IllegalRings` class extends `JavaPlugin` and orchestrates all ring handlers:

```java
public class IllegalRings extends JavaPlugin {
    private FlyRing flyRingHandler;
    private FireRing fireRingHandler;
    private WaterRing waterRingHandler;
    private HealRing healRingHandler;
    private PeacefullRing peacefulRingHandler;

    protected void setup() {
        // 1. Load config from mods/tiffy/config.json
        // 2. Initialize all ring handlers
        // 3. Register ECS systems with event manager
    }
}
```

**Key Points:**
- Config loads before handlers initialize (allows per-ring enable/disable)
- Each ring has an independent handler with its own lifecycle
- ECS systems registered via `getEventManager().subscribe()`

### Ring Handler Pattern

Each ring implements a common pattern for event-driven gameplay:

**FlyRing example:**
```java
public class FlyRing {
    private Set<Player> onlinePlayers;           // Track online players
    private Set<UUID> trackedFlightPlayers;      // Track who we gave flight to
    private ScheduledExecutorService scheduler;  // Background verification

    private void onInventoryChange(LivingEntityInventoryChangeEvent event) {
        // Check if ring was added/removed
        // Update flight state accordingly
    }

    private void protectRingWearersFromFallDamage() {
        // Runs every 500ms via scheduler
        // Resets Y-velocity for fall damage immunity
    }
}
```

**Pattern Details:**
- Event-driven (no ticking required)
- Thread-safe collections (`ConcurrentHashMap`, `CopyOnWriteArraySet`)
- Background schedulers for verification/protection loops
- Server world thread synchronization via `world.execute()`

### ECS-Based Damage Systems

Rings that provide immunity use the Hytale ECS system:

**FireRingDamageSystem example:**
```java
public class FireRingDamageSystem extends EntityEventSystem<DamageEvent> {
    @Override
    public void execute(Entity entity, DamageEvent event) {
        if (isWearingFireRing(entity)) {
            event.setCancelled(true);  // Cancel heat-based damage
        }
    }
}
```

**Key Design:**
- Registered in high-priority `FilterDamageGroup`
- Only cancels specific damage types (fire, lava, magma, burn)
- Preserves other hazards (physical, fall - handled by FlyRing)

### Configuration System (ModConfig.java)

Config is loaded from `mods/tiffy/config.json`:

```java
public class ModConfig {
    public static class RingEnabled {
        public boolean flyRing = true;
        public boolean fireRing = true;
        // ... etc
    }

    public static class GameplayValues {
        public double lifestealPercent = 0.35;  // 35% by default
    }

    public static class Config {
        public RingEnabled enabled;
        public GameplayValues gameplay;
        public boolean debugLogging = false;
    }
}
```

**Features:**
- Auto-creates config with defaults on first run
- Auto-updates existing config with new fields (via GSON + save cycle)
- Accessed via `ModConfig.getInstance()` after loading
- Per-ring enable/disable flags (disables ring effects without recompile)
- Configurable game values (lifesteal %)

### Logging Helper (Log.java)

Conditional logging to reduce spam:

```java
public class Log {
    public static void info(JavaPlugin plugin, String message) {
        // Only logs if config.debugLogging = true
    }

    public static void severe(JavaPlugin plugin, String message) {
        // Always logs (stability monitoring)
    }

    public static void setup(JavaPlugin plugin, String message) {
        // Always logs (initialization feedback)
    }
}
```

**Usage Pattern:**
```java
Log.setup(this, "Ring initialized!");          // Always shown
Log.info(this, "Player equip detected");       // Only if debugLogging=true
Log.severe(this, "Critical error occurred!");  // Always shown
```

## Thread Safety & Synchronization

### Thread-Safe Collections

All player/UUID tracking uses concurrent structures:

```java
private Set<Player> onlinePlayers = ConcurrentHashMap.newKeySet();
private Map<UUID, Long> flightGrantTime = new ConcurrentHashMap<>();
```

### World Thread Synchronization

Component access (inventory, velocity, movement) is scheduled on Hytale's world thread:

```java
world.execute(() -> {
    // Safe to access/modify player components here
    player.getComponent(Velocity.class).setY(0);
});
```

**Critical:** Never access player components directly from event handlers - always schedule on world thread.

## Adding a New Ring

### 1. Create the Ring Handler Class

```java
public class MyRing {
    private final JavaPlugin plugin;

    public MyRing(JavaPlugin plugin) {
        this.plugin = plugin;
        setup();
    }

    private void setup() {
        // Register event listeners
        // plugin.getEventManager().subscribe(MyEvent.class, this::onMyEvent);
    }

    private void onMyEvent(MyEvent event) {
        // Ring logic here
    }
}
```

### 2. Add Config Option

In `ModConfig.java`, add to `RingEnabled`:
```java
public boolean myRing = true;
```

### 3. Register in IllegalRings.java

```java
private MyRing myRingHandler;

protected void setup() {
    // ... other initialization ...

    myRingHandler = new MyRing(this);

    // Register ECS systems if needed
    if (ModConfig.getInstance().enabled.myRing) {
        getEventManager().subscribe(MyEventSystem.class, myRingHandler::onMyEvent);
    }
}
```

### 4. Add Item Definition

Create `resources/Server/Item/Items/Tool/Jewelry/Jewelry_My_Ring.json`:
```json
{
  "ID": "Jewelry_My_Ring",
  "Name": "My Ring",
  "Icon": "Jewelry_My_Ring",
  "Rarity": "Legendary",
  "Description": "A magical ring that does X"
}
```

### 5. Add Icon

Place PNG image at `resources/Common/Icons/ItemsGenerated/Jewelry_My_Ring.png`

### 6. Add Item Name

Add to `resources/Server/Languages/en-US/items.lang`:
```
item.Jewelry_My_Ring=My Ring
```

### Modifying Ring Behavior

Most gameplay adjustments can be made in config without recompile:
- Enable/disable individual rings
- Adjust lifesteal percentage
- Toggle debug logging

For code changes (damage types, immunity conditions, etc.), edit the corresponding ring class and rebuild.

### Item Registration

Items are registered via JSON files in resources. Hytale auto-discovers items from manifest-specified asset packs. No explicit Java registration needed (unlike energy blocks).

## Known Limitations

- **No persistence:** Ring state not saved across server restarts (design choice for simplicity)
- **Config hot-reload:** Not supported; requires server restart to apply changes
- **Asset pack delivery:** Icons/items depend on Hytale's asset discovery (no custom loader)
- **ECS system registration:** Manual subscription in IllegalRings.java (not auto-discovered)
